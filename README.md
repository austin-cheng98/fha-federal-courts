# Measuring Fair Housing Act Signals in Federal District-Court Opinions

Code and data for the paper. The pipeline assembles a canonicalized retrieval of the CourtListener federal NOS-443 (housing civil-rights) opinion-cluster population, extracts claim indicators, doctrinal frameworks, directional outcome cues, and a rule-based legal-signal breadth measure with hand-validated NLP. The released corpus is a district-court opinion corpus aggregated by circuit geography; it is not an appellate-law corpus. The housing merge is a feasibility diagnostic, not a reported causal or fixed-effects estimate. Synthetic data are estimator checks only.

## Repository layout

```
src/fha/      Pipeline package, one module per methodology stage
scripts/      Command-line entry points (run, harvest, validation, paper assets)
tests/        Offline correctness tests (key test recovers BETA_TRUE)
data/         raw / processed / external inputs, and the validation gold set
outputs/      Generated tables, figures, models (created on run)
```

## Installation

Python 3.11+ (tested through 3.14). No GPU required.

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e .          # or: pip install -r requirements.txt
```

Core dependencies: numpy, pandas, scipy, scikit-learn, statsmodels, requests, matplotlib. A legal-BERT embedding backend (Step 4) is optional and requires torch/transformers.

## Quickstart

```bash
# 1. Validate the estimators offline (no keys, no network)
PYTHONPATH=src python -m pytest -q
python scripts/run_pipeline.py --source synthetic   # estimator plumbing check

# 2. Reproduce the paper's figures and tables from the frozen inputs
python scripts/make_paper_assets.py                 # reads data/processed/paper_corpus.jsonl

# 3. Reproduce extraction validation and robustness
python scripts/score_goldset.py
python scripts/validation_robustness.py

# 4. (Optional) re-harvest from CourtListener — rate-limited, resumable
export COURTLISTENER_TOKEN=...                       # free token; unlocks full text
python scripts/collect_live.py
```

## Data

Repository-relative paths. The shipped inputs reproduce the paper with no network or secret:

- `data/processed/paper_corpus.jsonl` — 751 canonical-NOS-443 full-text opinion clusters.
- `data/raw/bulk_fha_cases.jsonl` — 757 canonical-NOS-443 opinion-cluster metadata rows.
- `data/validation/excluded_non_nos443.jsonl` — 180 candidate rows excluded by the exact NOS-443 normalizer.
- `data/external/housing_panel.csv` — circuit-by-year housing panel: Census ACS Black-White dissimilarity (the estimated outcome) plus an HMDA denial-rate column (not used as a within-unit outcome).
- `data/validation/` — the 93-opinion enriched/choice-based hand-coded overlap (30 double-coded) and the validation/robustness outputs.

Derived tables (case features, FEII/estimation panels, doctrine maps, synthetic data) are regenerated by the scripts and not committed. The CourtListener bulk dump, large synthetic corpora, logs, and secrets are excluded. See [`data/README.md`](data/README.md) for the full map.

Court opinions are U.S. government works in the public domain, redistributed here in derived, machine-readable form.

## Methodology

Each pipeline stage maps to a module in `src/fha/`; validation and robustness are driven by scripts.

| Stage | Module |
|---|---|
| Court-to-circuit map, FHA citation detection | `reference.py` |
| Paths and secret resolution | `config.py` |
| CourtListener REST v4 client (optional live harvest) | `courtlistener.py` |
| Bulk-dump streaming (how the corpus was built) | `bulkdata.py` |
| NOS-443 rule-based FHA identification | `classify.py` |
| Claims / frameworks / disposition / legal-signal extraction | `extract.py` |
| Doctrinal regimes (TF-IDF + LSA + K-means) | `doctrine.py` |
| Federal Enforcement Intensity Index | `feii.py` |
| Census dissimilarity + HMDA | `housing.py` |
| Optional TWFE + wild-cluster bootstrap (synthetic/feasible panels) | `econometrics.py` |
| Placebo / region-FE robustness | `robustness.py` |
| Extraction validation vs the gold set (precision/recall/F1/kappa) | `scripts/score_goldset.py` |
| Robustness suite (negation, FEII leave-one-out, era, regime terms, case-mix) | `scripts/validation_robustness.py` |
| Validation DGP | `synth.py` |
| Orchestrator | `pipeline.py` |
| Report and summary writers | `reports.py` |

The FEII is a descriptive circuit-year index: an equal-weight z-scored composite of log opinion-cluster volume, shrunken outcome cues, and normalized remedy-cue intensity. The housing input uses non-overlapping ACS vintages (2012, 2017, and 2022). Only eight substantive-corpus cells match, all in 2022, so the real-data branch records a feasibility note rather than a fixed-effects coefficient, bootstrap p-value, event study, or DiD estimate. Synthetic data retain the seeded estimator check.
